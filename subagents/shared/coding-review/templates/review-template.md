## 2.1 计划外变更识别

> 识别 git diff 中存在但 plan 未描述的变更。判断是合理的补充还是偏差。

| 变更文件/模块 | 变更内容摘要 | 判定 | 备注 |
|----------|----------|------|------|
| {文件} | {改了什么} | 接受 / 需确认 / 回退 | {说明} |

**判定规则**：
- **计划内** — 已经同步写入 plan 的需求变动，按实施步骤审查，不列为计划外偏差
- **接受** — 合理的补充（补测试、修 bug、完善边界），记录后接受，不建议恢复
- **需确认** — 涉及逻辑变更或新增功能，需与用户确认是否符合预期，确认前不建议恢复
- **回退** — 明确无关、有害、破坏计划目标或引入风险，才建议恢复为 plan 描述的实现
